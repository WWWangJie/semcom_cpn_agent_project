# 项目技术描述

## 项目名称

面向 6G 语义通信与算力网络融合的 Token-aware Agent 可靠性仿真系统

## 项目背景

在 6G 语义通信与算力网络融合场景中，传统通信网络通常以比特传输、带宽分配、CPU 频率分配为核心建模对象。然而，在大模型和语义通信融合的网络中，语义信息的压缩、传输、恢复和计算过程更适合使用 Token 进行统一抽象。因此，本项目构建了一套面向 Token 资源分配、FBL 物理层传输和终端语义恢复的端到端仿真系统。

## 做了什么

本项目实现了以下能力：

1. 构建了基于 Token 的语义任务模型，将终端任务表示为不同复杂度、不同紧急程度和不同 Token 需求的语义任务；
2. 构建了 FBL 物理层传输可靠性模型，将有限块长误块率与 Token 数量、信道状态、块长等因素关联；
3. 构建了 Token-语义准确率非线性恢复模型，用饱和型函数描述 Token 数量对语义恢复准确率的影响；
4. 构建了算力网络中的边缘节点模型，用 Token/s 表征边缘节点处理语义 Token 的能力；
5. 设计了轻量级 Token-aware Agent 调度器，综合考虑物理层可靠性、语义准确率、算力节点负载和时延约束；
6. 实现了随机调度、信道贪婪调度和 Agent 调度三类策略，用于系统级对比；
7. 输出平均端到端成功概率、端到端误差概率、FBL 误块率、语义准确率、Token 数、时延和任务截止期满足率等指标。

## 技术路线

项目采用 Python 构建，核心技术包括：

- `dataclass`：定义任务、终端、边缘节点、信道状态和调度决策；
- `numpy`：完成随机信道、任务生成和数值计算；
- `pandas`：完成仿真结果汇总和 CSV 输出；
- `matplotlib`：绘制调度策略对比图；
- 有限块长近似公式：完成物理层可靠性建模；
- Token 非线性收益函数：完成语义恢复准确率建模；
- 启发式 Agent Score：完成多目标资源调度决策。

## 达到了什么效果

该项目能够形成一个完整的“任务生成—Token 分配—物理层传输—边缘计算—语义恢复—可靠性统计”的端到端仿真闭环。通过对比基线调度策略，能够展示 Token-aware Agent 在低时延、高可靠性语义通信场景中的潜在优势。

该项目适合作为论文实验原型、科研汇报 Demo、AI Agent 辅助通信建模案例，以及后续扩展强化学习调度、真实语义模型拟合和多跳 6G 网络机制的基础代码框架。
